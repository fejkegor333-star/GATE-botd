# Интеграция модулей

## Потоки данных

### 1. Обнаружение нового листинга → Торговля

```
[Monitoring] → [WebSocket] → [Trading] → [Risk] → [Telegram]
     ↓              ↓            ↓          ↓           ↓
  Новый листинг  Стакан    Позиция   Проверка  Уведомление
```

**Пошагово:**
1. `monitor.py` обнаруживает новый контракт → сохраняет в БД
2. `websocket_client.py` подписывается на стакан нового контракта
3. Анализ стакана определяет сигнал на покупку
4. `risk_manager.py` проверяет баланс и лимиты
5. `position_manager.py` открывает позицию
6. `notifier` отправляет уведомление в Telegram

### 2. Управление открытой позицией

```
[WebSocket] → [Trading] → [Risk] → [Telegram]
     ↓            ↓          ↓           ↓
  Обновление    Усреднение  Проверка  Уведомление
  цены
```

**Пошагово:**
1. WebSocket обновляет стакан
2. Обновляем цену позиции
3. Проверяем сигналы усреднения (-3%, -7%, -10%)
4. Проверяем сигналы закрытия (TP +50%, SL -10%, Timeout)
5. Риск-менеджер проверяет лимиты перед усреднением
6. Отправляем уведомление о действиях

### 3. Закрытие позиции

```
[Trading] → [Risk] → [DB] → [Telegram]
    ↓         ↓       ↓         ↓
  Закрытие  PnL    Запись  Уведомление
```

**Пошагово:**
1. Сигнал закрытия (TP/SL/Timeout)
2. Закрываем позицию, вычисляем PnL
3. `risk_manager.record_trade_pnl()` - записываем PnL
4. Проверяем дневной лимит потерь
5. Сохраняем торговую операцию в БД
6. Отправляем финальное уведомление

## Защита от ошибок

### Rate Limiting
```python
# Все API запросы через защиту
result = await risk_manager.execute_with_protection(api_call)
```

### Circuit Breaker
- После 3 ошибок → блокировка на 60 сек
- Автоматическое восстановление
- Защита от каскадных сбоев

### Проверка перед каждой сделкой
```python
can_trade, reason = await risk_manager.check_before_trade(100.0)
if not can_trade:
    logger.warning(f"Торговля запрещена: {reason}")
    return
```

Проверяет:
- Баланс > $100
- Достаточно средств (с запасом 20%)
- Дневной PnL > -$1000
- Предохранитель не открыт

## Запуск бота

### Основной цикл

```python
# main.py
async def main():
    # 1. Валидация конфигурации
    config.validate()

    # 2. Инициализация БД
    db.init_db()

    # 3. Запуск торгового бота
    await bot.start()
    #    ├── Загрузка активных позиций
    #    ├── Запуск мониторинга листингов
    #    ├── Подключение WebSocket
    #    └── Запуск фоновых задач:
    #        ├── WebSocket listen loop
    #        ├── Positions monitor loop (каждые 5 сек)
    #        └── Cleanup loop (каждую минуту)
```

### Фоновые задачи

**WebSocket Loop:**
- Прослушивает обновления стаканов
- Обрабатывает сигналы на покупку
- Обновляет цены позиций
- Проверяет сигналы закрытия

**Positions Monitor Loop (каждые 5 сек):**
- Проверяет все открытые позиции
- Проверяет сигналы усреднения
- Обрабатывает усреднения

**Cleanup Loop (каждую минуту):**
- Закрывает просроченные позиции (> 24 часа)
- Очищает старые данные

## Аварийное восстановление

### Если бот упал
1. Проверьте логи: `logs/bot.log`
2. Активные позиции сохранены в БД
3. При перезапуске позиции загрузятся автоматически
4. WebSocket переподключится автоматически
5. Circuit Breaker восстановится через 60 сек

### Ручное закрытие позиций
```python
from src.trading.trader import position_manager

# Закрыть все позиции
positions = position_manager.get_all_positions()
for symbol, pos in positions.items():
    await position_manager.close_position(
        symbol,
        pos.current_price or pos.entry_price,
        reason='manual'
    )
```

## Производительность

### Оптимизация
- WebSocket используется вместо REST API (быстрее)
- Ленивая загрузка Telegram бота
- Асинхронные операции (asyncio)
- Пулирование соединений к БД

### Нагрузка
- Monitoring: 1 запрос/5 сек
- WebSocket: постоянное соединение
- Positions check: каждые 5 сек
- Cleanup: каждую минуту
- Telegram: только при событиях
